Web Programming & Interactivity with JavaScript

A 2-hour introduction for researchers building interactive web content

Overview

This workshop introduces JavaScript as the programming layer of the web. It is designed for researchers, librarians, and research staff who already have basic familiarity with HTML and CSS and want to add interactivity to research pages.

  • Duration: 2 hours
  • Prerequisites: Basic HTML & CSS
  • Outcome: Added interactivity to webpage

    JavaScript Basics

    JavaScript is a programming language that runs in the browser.

    Variables

    Declare variables using let, const (for contants), or (older) var.

    // A simple variable
let projectTitle = "Climate Data Explorer";

// Logging output
console.log(projectTitle);

    To see console.log information: In Chrome - click View>Developer>Developer Tools, then select Console.

    Variable names must:

    • Start with a letter, _, or $
    • Contain only letters, numbers, _, or $
    • Not start with a number
    • Not be a reserved keyword

    JavaScript is case-sensitive. So variables named 'age' and 'Age' are different. Best to keep varaibles

    Conditionals

    if

    if / else

    
const isLoggedIn = false;

  console.log("Grade: B");

  console.log("Grade: C");

  console.log("Grade: F");

    Loops

    for loop (classic)

    
for (let i = 0; i < 5; i++) {
    console.log(i);
}

    Looping through an array with for

    
const colors = ["red", "green", "blue"];

for (let i = 0; i < colors.length; i++) {
  console.log(colors[i]);
}

    for...of (modern & clean)

    Best for looping over values in arrays and strings. 
    
const fruits = ["apple", "banana", "orange"];

for (const fruit of fruits) {
  console.log(fruit);
}

    for...in (objects)

    
const user = {
  name: "Kevin",
  role: "Researcher"
};

for (const key in user) {
  console.log(key, user[key]);
}

    while loop

    Runs while a condition is true. 
    
let count = 0;

while (count < 3) {
  console.log(count);
  count++;
}

    do...while

    Runs at least once, even if the condition is false. 
    
let num = 5;
do {
  console.log(num);
  num++;
} while (num < 3);

    break and continue

    
for (let i = 1; i <= 5; i++) {
  if (i === 3) {
    continue; // skip 3
  }

  if (i === 5) {
    break; // stop loop
  }

  console.log(i);
}

    The DOM: Connecting JavaScript to HTML

    The Document Object Model (DOM) allows JavaScript to read and change HTML.

    To make sure the HTML elements are present before the JavaScript runs, be sure to wait for the 'DOMContentLoaded' event to fire.

    // Select an element
document.addEventListener('DOMContentLoaded', () => {
    // Select an element
    const heading = document.querySelector("h1");
    console.log(heading)
    // Change its text
    heading.textContent = "Interactive Research Page";

});

    Think of the DOM as a bridge between your HTML structure and JavaScript logic.

    Events & User Interaction

    Events allow your page to respond to user actions such as clicks or typing. The following example shows how to show/hide an element:

    HTML

    <button id="toggle">Show Abstract</button>
<p id="abstract" hidden>
  This project examines...
</p>

    JavaScript

    const button = document.querySelector("#toggle");
const abstract = document.querySelector("#abstract");

button.addEventListener("click", () => {
  abstract.hidden = !abstract.hidden;
});

    Forms

    Accepting user input is often handled using a form element. The following example shows how to create a simple calculator using html and javascript.

    HTML

    JavaScript

    
function calculate(event) {
      event.preventDefault(); // stop form from refreshing page

      const equation = document.getElementById("equation").value;
      const resultEl = document.getElementById("result");

      // Note: eval() is dangerous in real apps.
      // Expressions should be sanitized before processing to prevent injecting Javascript.
      try {
        const answer = eval(equation);
        resultEl.textContent = answer;
      } catch (error) {
        resultEl.textContent = "Invalid expression";
      }
    }
